# Dockerfile syntax
 - one instruction per line
 - continuation of lines with \
 - `INSTRUCTION arguments`
 - `# Comments`



## INSTRUCTIONS
 - `ADD`, `ARG`, `CMD`, `COPY`, `ENTRYPOINT`, `ENV`
 - `EXPOSE`, `FROM`, `HEALTHCHECK`, `LABEL`, `ONBUILD`
 - `RUN`, `SHELL`, `STOPSIGNAL`, `USER`, `VOLUME`, `WORKDIR`
